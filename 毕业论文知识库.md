## 2. 创新点一：自适应EMA滤波

### 2.1 算法原理

**指数滑动平均（Exponential Moving Average, EMA）** 是一种一阶低通滤波器，相比滑动平均具有更好的实时性和内存效率。

**传统滑动平均**：
```
y[n] = (x[n] + x[n-1] + ... + x[n-N+1]) / N
```
- 需要保存N个历史样本
- 所有样本等权重

**EMA滤波**：
```
y[n] = α × x[n] + (1-α) × y[n-1]
```
- 只需保存上一时刻输出
- 历史数据指数衰减权重

**参数α的含义**：
- α越大，当前样本权重越高，响应越快
- α越小，历史样本权重越高，滤波越平滑

### 2.2 自适应策略

本系统根据**运动速度**动态调整α值：

| 运动状态 | 条件 | α值 | 效果 |
|---------|------|-----|------|
| 慢速运动 | 速度 < 15像素/帧 | 0.05 | 深度平滑，减少抖动 |
| 快速运动 | 速度 ≥ 15像素/帧 | 0.15 | 快速响应，减少延迟 |

### 2.3 代码实现

**文件**：`svm/HandTrackingModule.py`

self.ema_y = None

# 历史速度用于计算加速度

## 3. 创新点二：二阶离群点检测

### 3.1 问题背景

MediaPipe手部检测在快速运动或光照变化时可能出现**跳变**（单帧坐标突然偏移几十像素），这会：
1. 破坏EMA滤波的平滑效果
2. 污染速度历史数据
3. 导致预测控制失效

### 3.2 一阶检测：坐标突变

检测当前坐标与EMA滤波输出的偏差：
```python
deviation_x = abs(thumb_tip_x - detector.ema_x)
deviation_y = abs(thumb_tip_y - detector.ema_y)
outlier_threshold = 50  # 像素

if deviation_x > outlier_threshold or deviation_y > outlier_threshold:
    alpha = alpha * 0.3  # 降低alpha，保守更新
```

### 3.3 二阶检测：速度突变

检测当前速度与历史平均速度的比值：
```python
current_velocity = abs(velocity_x) + abs(velocity_y)
if len(detector.velocity_x_history) >= 3:
    history_avg = sum(detector.velocity_x_history) + sum(detector.velocity_y_history)
    history_avg = history_avg / (len(detector.velocity_x_history) + len(detector.velocity_y_history)) * 2
    if history_avg > 5 and current_velocity > history_avg * 3:
        alpha = alpha * 0.2  # 极度保守
```

**判断逻辑**：
- 如果历史平均速度 > 5像素/帧（排除静止干扰）
- 且当前速度 > 历史平均的3倍
- 则判定为跳变，使用极小α值(0.2倍)

### 3.4 代码实现

**文件**：`svm/HandTrackingModule.py`，第226-241行

```python
# 一阶离群点检测：单帧坐标突变

deviation_y = abs(thumb_tip_y - detector.ema_y)
outlier_threshold = 50

    alpha = alpha * 0.3

# 二阶离群点检测：速度突变（当前速度远超历史平均）

    history_avg = history_avg / (len(detector.velocity_x_history) + len(detector.velocity_y_history)) * 2
    if history_avg > 5 and current_velocity > history_avg * 3:
        alpha = alpha * 0.2  # 极度保守
```

## 4. 创新点三：线性预测控制

### 4.1 问题背景

系统存在固有延迟：
1. 摄像头采集延迟：~33ms（30fps）
2. MediaPipe处理延迟：~20ms
3. 串口传输延迟：~1ms
4. PID控制响应延迟

总计约50-60ms滞后，导致**云台响应总是"慢半拍"**。

### 4.2 预测控制原理

**前馈控制思想**：不等待误差产生，而是根据历史轨迹预测未来状态，提前补偿。

**线性外推预测**：
```
v[n] = y[n] - y[n-1]          # 当前速度
ŷ[n+1] = y[n] + v[n]         # 预测下一帧位置
```

即：**预测位置 = 当前平滑位置 + 速度**

### 4.3 可靠性保障

| 保护机制 | 阈值 | 作用 |
|---------|------|------|
| 速度过小不预测 | <2像素/帧 | 防止静止时噪声被放大 |
| 预测偏差过大回退 | >80像素 | 预测不可靠时使用平滑值 |

```python
# 计算预测速度
pred_velocity_x = detector.ema_x - detector.last_ema_x
pred_velocity_y = detector.ema_y - detector.last_ema_y

# 预测下一帧位置
pred_x = detector.ema_x + pred_velocity_x
pred_y = detector.ema_y + pred_velocity_y

# 预测合理性检测
pred_deviation = abs(pred_x - detector.ema_x) + abs(pred_y - detector.ema_y)
if pred_deviation > detector.prediction_tolerance:
    pred_x = detector.ema_x  # 回退到平滑值
    pred_y = detector.ema_y
    detector.use_prediction = False

# 速度太小也不预测（防噪声放大）
if abs(pred_velocity_x) < detector.min_velocity_for_prediction and \
   abs(pred_velocity_y) < detector.min_velocity_for_prediction:
    pred_x = detector.ema_x
    pred_y = detector.ema_y
```

### 4.4 代码实现

**文件**：`svm/HandTrackingModule.py`，第254-284行

## 5. 创新点四：手势状态机

### 5.1 问题背景

手势识别存在**误触发**问题：
1. 快速挥手时，同一手势可能被误判为多个不同手势
2. 手在摄像头边缘时，检测不稳定
3. SVM分类器在边界情况下可能跳变

### 5.2 状态机设计

```
┌──────────┐    手势出现     ┌──────────────┐
│ WAITING  │ ─────────────▶ │ CONFIRMING   │
│  (等待)  │                │   (确认中)    │
└──────────┘                └───────┬───────┘
      ▲                            │
      │                            │ 保持0.5秒且手势一致
      │ 手势变化                  ▼
      │                      ┌──────────────┐
      │◀────────────────────│  TRIGGERED   │
      │                      │   (已触发)    │
      │                      └───────┬───────┘
      │                              │
      │          冷却1秒后            │
      └──────────────────────────────┘
```

### 5.3 参数说明

| 参数 | 值 | 说明 |
|------|-----|------|
| hold_duration | 0.5秒 | 手势保持确认时间 |
| cooldown_duration | 1.0秒 | 触发后冷却时间 |

### 5.4 代码实现

**文件**：`svm/HandTrackingModule.py`

```python

self.last_gesture = None      # 上一次检测到的手势
self.confirmed_gesture = None # 已确认的手势
self.hold_start_time = None   # 开始保持的时间戳
self.hold_duration = 0.5      # 触发需要的保持时间（秒）

```

```python
# ========== 手势状态机 ==========
cTime = time.time()

if detector.gesture_state == "WAITING":
    if current_gesture is not None:
        detector.last_gesture = current_gesture
        detector.hold_start_time = cTime
        detector.gesture_state = "CONFIRMING"

elif detector.gesture_state == "CONFIRMING":
    if current_gesture == detector.last_gesture:
        if cTime - detector.hold_start_time >= detector.hold_duration:
            detector.confirmed_gesture = current_gesture
            detector.gesture_state = "TRIGGERED"
            detector.cooldown_start = cTime
    else:
        detector.gesture_state = "WAITING"
        detector.last_gesture = None
        detector.hold_start_time = None

elif detector.gesture_state == "TRIGGERED":
    if cTime - detector.cooldown_start >= detector.cooldown_duration:
        detector.gesture_state = "WAITING"
        detector.confirmed_gesture = None
```

## 6. 固件层：增量式PID控制

### 6.1 增量式PID原理

**位置式PID**：
```
u[n] = Kp·e[n] + Ki·Σe[i] + Kd·e[n]-e[n-1]
```

**增量式PID**：
```
Δu[n] = u[n] - u[n-1]
       = Kp·(e[n] - e[n-1]) + Ki·e[n] + Kd·(e[n] - 2e[n-1] + e[n-2])
u[n] = u[n-1] + Δu[n]
```

**优点**：
- 计算量小，适合嵌入式
- 增量输出，避免积分饱和
- 抗干扰能力强

### 6.2 代码实现

**文件**：`control/HARDWARE/PID/pid.c`

```c
// 增量式PID
int incre_pid(int present, u16 target, PID_TypeDef* PID)
{
    PID->error = target - present;

    PID->p_out = PID->Kp * (PID->error - PID->last_error);
    PID->i_out += PID->Ki * PID->error;
    PID->d_out = PID->Kd * (PID->error - 2*PID->last_error + PID->last_error2);

    PID->output += PID->p_out + PID->i_out + PID->d_out;

    PID->last_error2 = PID->last_error;
    PID->last_error = PID->error;

    return PID->output;
}
```

### 6.3 PID参数整定

| 轴 | Kp | Ki | Kd | 特点 |
|----|----|----|----|------|
| X | 0.04 | 0 | 0.30 | 低比例，高微分，抑制抖动 |
| Y | 0.05 | 0 | 0.30 | 略高比例，响应更快 |

### 6.4 主控制循环

```c
while(1)

% 增量式PID
Δu[n] = Kp·(e[n] - e[n-1]) + Ki·e[n] + Kd·(e[n] - 2e[n-1] + e[n-2])

% PWM频率
f = 72MHz / ((ARR+1)·(PSC+1))
```
